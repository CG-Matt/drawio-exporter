# Drawio Exporter

Self-contained CLI wrapper for exporting `.drawio` files to PNG/SVG using the draw.io `export3` renderer.

## Layout

- `drawio-exporter.py`: export command
- `vendorize.py`: fetches/copies draw.io webapp assets into `.webapp/`
- `.webapp/`: vendored runtime assets used for export

## Requirements

- Python 3.10+
- Playwright Chromium runtime

## Setup

### Linux/macOS

Quick setup script (supports venv/global selection):

```bash
./setup.sh
```

Non-interactive:

```bash
./setup.sh --mode venv
# or
./setup.sh --mode global
```

Manual setup (equivalent to `--mode venv`):

```bash
python3 -m venv .venv
.venv/bin/python -m pip install playwright
.venv/bin/python -m playwright install chromium
```

### Windows

Command Prompt quick setup script (supports venv/global selection):

```bat
setup.bat
```

Command Prompt non-interactive:

```bat
setup.bat --mode venv
# or
setup.bat --mode global
```

Manual setup (equivalent to `--mode venv`):

```powershell
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install playwright
.\.venv\Scripts\python.exe -m playwright install chromium
```

## Vendorize draw.io assets

### Linux/macOS

From local source:

```bash
.venv/bin/python vendorize.py --source-root /path/to/drawio
```

From GitHub:

```bash
.venv/bin/python vendorize.py --github
```

Pin GitHub ref:

```bash
.venv/bin/python vendorize.py --github --ref <tag-or-branch-or-commit>
```

Full fallback set instead of minimal discovered set:

```bash
.venv/bin/python vendorize.py --source-root /path/to/drawio --full
# or
.venv/bin/python vendorize.py --github --full
```

### Windows

From local source:

```powershell
.\.venv\Scripts\python.exe .\vendorize.py --source-root C:\path\to\drawio
```

From GitHub:

```powershell
.\.venv\Scripts\python.exe .\vendorize.py --github
```

Pin GitHub ref:

```powershell
.\.venv\Scripts\python.exe .\vendorize.py --github --ref <tag-or-branch-or-commit>
```

Full fallback set instead of minimal discovered set:

```powershell
.\.venv\Scripts\python.exe .\vendorize.py --source-root C:\path\to\drawio --full
# or
.\.venv\Scripts\python.exe .\vendorize.py --github --full
```

## Export diagrams

### Linux/macOS

```bash
.venv/bin/python drawio-exporter.py --input /path/to/file.drawio --output ./out.png --format png
.venv/bin/python drawio-exporter.py --input /path/to/file.drawio --output ./out.svg --format svg
```

### Windows

```powershell
.\.venv\Scripts\python.exe .\drawio-exporter.py --input C:\path\to\file.drawio --output .\out.png --format png
.\.venv\Scripts\python.exe .\drawio-exporter.py --input C:\path\to\file.drawio --output .\out.svg --format svg
```

Multi-page files export one output per page with page suffixes.

## Notes

- Exporting requires `.webapp/` to exist (generated by `vendorize`).
- Default scale is `10` (`-s` / `--scale`).

## Third-party dependencies

- draw.io / diagrams.net webapp assets: Apache-2.0.
  Source: https://github.com/jgraph/drawio
- Playwright (Python): Apache-2.0.
  Source: https://github.com/microsoft/playwright-python

When using `vendorize.py --github`, assets are fetched from upstream draw.io at the selected ref.
